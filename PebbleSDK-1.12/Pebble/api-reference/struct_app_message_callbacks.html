void(* AppMessageCallbacks::in_dropped)(void *context, AppMessageResult reason)

When an incoming message is dropped, this callback is called, to notify the app of the drop.

Parameters
contextPointer to application data as specified when registering the callback.
reasonThe reason why the message was dropped Can be APP_MSG_BUSY or APP_MSG_BUFFER_OVERFLOW
void(* AppMessageCallbacks::in_received)(DictionaryIterator *received, void *context)
Parameters
receivedThe dictionary iterator to the received message.
contextPointer to application data as specified when registering the callback.
Note
The dictionary iterator can be used during the callback only. After the callback has returned, the iterator will no longer be valid.
void(* AppMessageCallbacks::out_failed)(DictionaryIterator *failed, AppMessageResult reason, void *context)

Called afer an outgoing message failed to get sent.

Parameters
failedThe dictionary iterator to the message that failed to get sent.
reasonThe reason why it failed sending. Can be APP_MSG_SEND_TIMEOUT, APP_MSG_SEND_REJECTED, APP_MSG_NOT_CONNECTED, APP_MSG_APP_NOT_RUNNING, or the combination (APP_MSG_NOT_CONNECTED | APP_MSG_APP_NOT_RUNNING).
contextPointer to application data as specified when registering the callback.
Note
The dictionary iterator can be used during the callback only. After the callback has returned, the iterator will no longer be valid.
