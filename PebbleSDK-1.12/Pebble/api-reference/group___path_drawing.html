Drawing Paths
Graphics

Detailed Description

Functions to draw polygons into a graphics context.

Code example:

GPath my_path;
Layer my_layer;
static const GPathInfo BOLT_PATH_INFO = {
.num_points = 6,
.points = (GPoint []) {{21, 0}, {14, 26}, {28, 26}, {7, 60}, {14, 34}, {0, 34}}
};
// .update_proc of my_layer:
void my_layer_update_proc(Layer *my_layer, GContext* ctx) {
// Fill the path:
graphics_context_set_fill_color(ctx, GColorWhite);
gpath_draw_filled(ctx, &my_path);
// Stroke the path:
graphics_context_set_stroke_color(ctx, GColorBlack);
gpath_draw_outline(ctx, &my_path);
}
void setup_my_path(void) {
gpath_init(&my_path, &BOLT_PATH_INFO);
// Rotate 15 degrees:
gpath_rotate_to(&my_path TRIG_MAX_ANGLE / 360 * 15);
// Translate by (5, 5):
gpath_move_to(&my_path, GPoint(5, 5));
}
// For brevity, the setup of my_layer is not written out...

void gpath_move ( GPath *  path,
GPoint  delta 
)

Applies a relative offset to the path. The offset will be added to the current translation of the path.

Parameters
pathThe path onto which to apply the translation
deltaThe point which is used as the vector for the translation.
Note
Applying a translation does not affect the points in the path directly. The translation is applied on-the-fly during drawing, either using gpath_draw_filled() or gpath_draw_outline().

Data Structure Documentation

struct GPathInfo

Data structure describing a naked path.

Note
Note that this data structure only referes to an array of points; the points are not stored inside this data structure itself. In most cases, one cannot use a stack-allocated array of GPoints. Instead one often needs to provide longer-lived (static or "global") storage for the points.
Data Fields
int num_points The number of points in the points array.
GPoint * points Pointer to an array of points.
struct GPath

Data structure describing a path, plus its rotation and translation.

Note
See the remark with GPathInfo
