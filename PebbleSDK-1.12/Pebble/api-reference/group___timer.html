AppTimerHandle app_timer_send_event ( AppContextRef  app_ctx,
uint32_t  timeout_ms,
uint32_t  cookie 
)

Register a new timer to fire timeout_ms in the future. Applications will be notified of expired timers via a call to the app's timer_handler. Applications must register a PebbleAppTimerHandler with app_event_loop() in order to successfully use app timers.

Parameters
app_ctxOpaque application context reference, provided by the OS in the .init_handler. See PebbleAppHandlers.
timeout_msThe time interval after which the timer should fire and timer_handler should be called.
cookieThe cookie parameter can be used to differentiate between multiple active timers. The cookie value is passed into the timer_handler, for the app to inspect.
Returns
APP_TIMER_INVALID_HANDLE (0) on failure, an opaque timer handle on success. The returned handle can be used with app_timer_cancel_event() to cancel the timer.
